# vaultscout

Семантический поиск по вашей базе знаний Obsidian. Локально, без облаков.

Задайте вопрос своими словами — vaultscout найдёт релевантные фрагменты
заметок и даст ссылки на них. Эмбеддинги считаются локальной моделью:
содержимое базы не покидает вашу машину.

## Как это устроено

```
Obsidian vault → парсер (папки / frontmatter / заголовки) → chunking
    → локальные эмбеддинги → Postgres + pgvector → поиск (CLI)
```

- **Индексация**: парсер обходит vault, режет заметки на смысловые куски
  с учётом структуры (папки, заголовки, frontmatter), считает векторы
  локальной моделью и складывает в Postgres с pgvector.
- **Поиск**: CLI-команда «спроси» возвращает топ-N фрагментов со ссылками
  на исходные заметки.
- **Приватность**: никакие данные не отправляются во внешние сервисы.

## Конфигурация

Скопируйте `config.example.yaml` в `config.yaml` и укажите путь к вашему vault:

```yaml
vault:
  path: /path/to/your/obsidian/vault

embeddings:
  model: deepvk/USER-bge-m3   # локальная модель, работает на CPU
```

`config.yaml` не попадает в git — ваши пути и настройки остаются локальными.

## Требования

- Python 3.12+
- Docker (для Postgres + pgvector)
- ~2 ГБ RAM для модели эмбеддингов (GPU не нужен)
